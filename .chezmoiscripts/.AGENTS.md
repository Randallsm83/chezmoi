# .chezmoiscripts/

Auto-run scripts that fire during `chezmoi apply`. See [root AGENTS.md](../AGENTS.md) for chezmoi naming conventions, template variables, and the broader architecture.

## Run order

Chezmoi runs scripts alphabetically within each `before`/`pre`/`after` bucket, gated by the prefix:

```
run_before_*       → before any file is written
run_onchange_*     → re-runs when the rendered script body changes
run_once_*         → exactly once per machine (state-tracked)
run_after_*        → after files are written (default)
```

The two-digit numeric prefix (`00_`, `05_`, `55_`, `71_`, …) is a manual ordering knob — lower runs first. Reserved ranges as currently used:

| Range | Purpose |
|---|---|
| `00–05` | Backup, validate-secrets, shim repair |
| `35` | Terminfo / terminal data |
| `50` | Cross-app config emission (e.g. `dh-mise-toml`) |
| `55–59` | Network plumbing: librewolf, VPN-DNS routes, encrypted-DNS, unbound, browser-DoH, Caddy CA, DNS watcher |
| `70` | Editor extension sync (vscode) |
| `71` | AI tooling runtime config sync (claude-code, opencode) |
| (no prefix) | Catch-alls: `backup`, `install-packages`, `rebuild_bat_cache`, `generate_bat_themes`, `generate_opencode_themes`, `sync_claude_memories` |

## Platform pairing (CRITICAL)

Every cross-platform script ships as a `_windows.ps1.tmpl` + `.sh.tmpl` pair with **identical numeric prefix and base name**. `.chezmoiignore` excludes the wrong half per OS — DO NOT add a unix script without its Windows sibling (or vice versa) unless the operation is genuinely OS-exclusive (e.g. `*_windows.ps1.tmpl` for NRPT, registry, scoop shims).

OS-exclusive present today: scoop shim repair, mise LSP shim repair, `dh-mise-toml`, wezterm terminfo, unbound, vpn-dns-watcher, librewolf darwin/linux variants (3 separate files, not one `.sh.tmpl`).

## Re-running scripts

`run_once_*` and `run_onchange_*` are state-tracked. To force re-execution:

```bash
chezmoi state delete-bucket --bucket=scriptState
chezmoi apply
```

`run_onchange_*` self-invalidates when its rendered body hash changes — include `# checksum: {{ <data> | sha256sum }}` lines (or equivalent) at the top to make changes to source data re-trigger the script.

## Conventions

- **Shell**: `set -euo pipefail` (use `{{ template "common-header" . }}` for the canonical bash header).
- **PowerShell**: `$ErrorActionPreference = 'Stop'`; use `{{ template "ps-logging" . }}` for `Write-Status` / `Write-LogLine` / `Get-StatusLogFile`.
- **Idempotency**: scripts MUST be safe to re-run. Test additive paths, not destructive ones.
- **Gating**: wrap entire script body in `{{ if eq .chezmoi.os "windows" }}…{{ end }}` if Windows-only, or use feature flags (`{{ if .package_features.vscode }}…{{ end }}`).
- **Logging**: write to `$XDG_STATE_HOME/dotfiles/logs/<script>.log` (Unix) or via `Write-Status` (Windows) so failures are recoverable.

## Anti-patterns

- DO NOT call `op` directly here — secrets must come from `.secrets.*` resolved at init time. Each `op` call triggers a biometric prompt.
- DO NOT mutate `$HOME` files chezmoi manages — chezmoi owns them. Mutate state under `$XDG_STATE_HOME/$XDG_CACHE_HOME`.
- DO NOT add `run_once_*` for anything that needs to track source changes — use `run_onchange_*` instead.
